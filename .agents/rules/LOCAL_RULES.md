# Локальные правила проекта: Civitai Gallery Parser

## Управление версиями
- **Автоматический инкремент**: При каждом изменении функциональных файлов (`.js`, `.css`, `.html`, `.json`) необходимо увеличивать номер версии.
- **Местоположение**: Обновляйте версию одновременно в `manifest.json` и `package.json` для обеспечения синхронизации.
- **Стратегия версионирования**:
  - **Patch (x.x.Z)**: Исправление мелких ошибок, корректировка стилей или незначительные оптимизации. (напр., 1.3.1 -> 1.3.2)
  - **Minor (x.Y.x)**: Новые функции, новые компоненты UI или значительные обновления логики.
  - **Major (X.x.x)**: Масштабная переработка или изменения, нарушающие обратную совместимость.
- **Согласованность**: По завершении задачи версия должна отражать актуальное состояние проекта.

## Контроль рабочего процесса
- **Перечитывание правил**: После каждого изменения файла локальных правил (`LOCAL_RULES.md`) необходимо его ПЕРЕЧИТЫВАТЬ, чтобы гарантировать актуальность понимания стандартов. При этом агент ОБЯЗАН вывести в чат разницу (что было и что стало).
- **Язык правил**: Файл локальных правил (`LOCAL_RULES.md`) всегда должен быть на русском языке.
