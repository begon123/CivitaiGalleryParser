# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это

Расширение Chrome Manifest V3 («Civitai Gallery Parser»), которое собирает изображения и статистику реакций из пользовательских галерей на civitai.com. Чистые ES-модули — **без шага сборки, без бандлера, без транспиляции**. Корень репозитория *и есть* загружаемое расширение.

## Команды

- **Ручная загрузка/проверка**: `chrome://extensions/` → включить режим разработчика → «Загрузить распакованное расширение» → выбрать корень репозитория. После правок нажать иконку перезагрузки на карточке расширения.
- **Запуск тестов**: `npm run test-extension` (Playwright). Внимание: в репозитории сейчас нет ни тестовых файлов Playwright, ни конфигурации, поэтому команда ничего не делает, пока тесты не будут добавлены.

## Обязательное правило проекта: инкремент версии

Согласно `.agents/rules/LOCAL_RULES.md` (это первоисточник, на русском языке): любое изменение функционального файла (`.js`, `.css`, `.html`, `.json`) **обязано** сопровождаться повышением версии, и его нужно применять **одновременно** в `manifest.json` и `package.json` (держать их синхронными). Patch — исправления/стили, Minor — новые функции/UI, Major — масштабная переработка с нарушением совместимости. Сам файл правил должен оставаться написанным на русском языке.

## Архитектура

Три среды выполнения, взаимодействующие только через сообщения `chrome.runtime` и `chrome.storage.local`:

1. **`background.js`** — сервис-воркер. Переключает оверлей по клику на иконку в панели, повторно внедряет контент-скрипты, если они не загружены (`ensureContentScriptLoaded` сначала шлёт ping), и владеет потоком `save_data`.
2. **`content.js`** — внедряется на страницы civitai.com. Строит встроенный в страницу «Unified Overlay» и запускает сканер. Организован как обычные объекты-модули: `UI_CONTROLLER` (DOM оверлея + жизненный цикл), `SCROLLER` (автопрокрутка, обход модалок/NSFW-гейтов, определение простоя), `DATA_EXTRACTOR` (валидация изображений + статистика реакций).
3. **`results.html` + `results.js`** — отдельная страница расширения, открываемая в новой вкладке после сканирования. Отрисовывает галерею, фильтры и импорт/экспорт Excel.

`i18n.js` общий для всех сред и **должен загружаться раньше `content.js`** (см. порядок `content_scripts.js` в `manifest.json` и список внедрения в `background.js`). `excel_handler.js` и подключённая библиотека `lib/xlsx.full.min.js` загружаются только на странице результатов.

### Поток данных: сканирование → сохранение → результаты

- `content.js` сканирует, затем при закрытии оверлея (`UI_CONTROLLER.toggle`) отправляет `save_data` **только если `STATE.scanCompleted` равно true** — при отмене сканирования пользователем `collectedData` сбрасывается и ничего не отправляется.
- `background.js` обрабатывает `save_data`: копирует существующий `civitai_gallery_data_{author}` в `civitai_gallery_data_{author}_previous`, записывает новый массив в `civitai_gallery_data_{author}`, сохраняет `civitai_gallery_author`, затем открывает `results.html`.
- `results.js` (`compareScans`) сравнивает текущий и предыдущий наборы по `src` изображения и присваивает каждому элементу статус: `new`, `changed` (общее число реакций отличается, несёт `totalDelta`), `unchanged` или `removed`. При первом сканировании автора → всё помечается как `new`.

### Ключи chrome.storage.local

- `civitai_gallery_data_{author}` / `..._previous` — массивы текущего и предыдущего сканирования
- `civitai_gallery_author` — последний просканированный автор, используется страницей результатов, чтобы понять, что загружать
- `civitai_parser_lang` — язык интерфейса (`en` / `ru`)

### Внутренности парсинга (по своей природе хрупкие)

Все селекторы DOM civitai находятся в `SELECTOR_CONFIG`/`CONFIG` в начале `content.js` — если парсинг сломался, значит civitai изменил разметку; исправлять нужно в первую очередь там. Извлечение реакций многослойное: основной путь читает веб-компоненты civitai `<number-flow-react>` и парсит JSON из их атрибута `data`; запасные пути ищут эмодзи и сигнатуры SVG-иконок. Сканер намеренно удаляет модальные оверлеи и кликает кнопки NSFW/возрастного гейта (`CONFIG.MODAL_KEYWORDS`), чтобы разблокировать прокрутку, и останавливается досрочно после 3 подряд шагов прокрутки без новых изображений.

## Соглашения

- Добавление пользовательской строки: добавить ключ в **оба** блока `en` и `ru` в `i18n.js`, затем ссылаться на него через атрибуты `data-i18n` (HTML) или `I18N.t('key', params)` (JS). Подстановки `{param}` интерполируются методом `I18N.t`.
- Формат Excel: лист `Gallery Data` использует числовой столбец `Status` (0 — без изменений, 1 — новый, 2 — изменён); удалённые элементы попадают на отдельный лист `Removed Data`. Сопоставление при импорте/экспорте находится в `excel_handler.js`.
